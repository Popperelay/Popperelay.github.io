泛光的意义和实现原理

通俗地讲，泛光其实就是明亮区域发出的光晕效果。比如场景里面有一张白纸和一个白色光源(同样都是相同亮度的白色)，但是给白色光源加上一个光晕以后，光源会比白纸看起来感觉更亮。下图是虚幻引擎中的光晕效果：

光晕通常和HDR结合使用。下面我们来介绍一下光晕实现的原理和流程（核心原理是对明亮区域进行模糊处理，让它看起来有一种光晕效果）。如下图所示：

我们会像往常一样渲染HDR场景（下图中的左上角场景），在这个场景中诸如光源等物体是特别明亮的区域，我们把这些明亮区域提取出来（直接根据颜色值超过某个阈值来提取就可以了），获得下图中左下角所示的场景纹理，然后我们对这个纹理进行模糊处理（模糊结果就是中间下方那张图），模糊之后明亮区域就有了一种加上了光晕的感觉，然后我们把模糊后的纹理和原先的HDR场景纹理进行叠加，就能得到最终明亮区域带有光晕的场景（即最右边的那张图）。

提取亮色

要做的第一件事当然就是像上篇HDR文章一样，使用很亮的光源来照亮场景，然后将渲染结果存储到帧缓存的纹理附件上了。这个就不再赘述了。

接下来要做的就是提取出场景中颜色值超过某个阈值的明亮区域，我们可以使用一种叫做MRT（多渲染目标）的技术，在第一步渲染场景到纹理附件的同时，把场景中的明亮区域同时渲染到另一个纹理附件里（这样就只用一遍pass了）。而MRT其实就是在片元着色器中可以有多个输出的技术，我们只需要使用location这个布局标识符来声明就可以了，如下：

layout (location = 0) out vec4 FragColor_;
layout (location = 1) out vec4 BrightColor_;

不过它只在帧缓存确实添加了多个颜色缓冲附件时才有效，给同一个帧缓存添加多个纹理附件也很简单，我们只需要连续调用两次glFramebufferTexture2D就可以了：

GLuint ColorBufferTextures[2];
glGenTextures(2, ColorBufferTextures);
for (GLuint i = 0; i < 2; ++i)
{
    glBindTexture(GL_TEXTURE_2D, ColorBufferTextures[i]);
    glTexImage2D(GL_TEXTURE_2D, 0, GL_RGBA16F, WIDTH, HEIGHT, 0, GL_RGBA, GL_FLOAT, nullptr);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);  // we clamp to the edge as the blur filter would otherwise sample repeated texture values!
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);
    glBindTexture(GL_TEXTURE_2D, 0);
}

GLuint DepthRenderBuffer = -1;
glGenRenderbuffers(1, &DepthRenderBuffer);
glBindRenderbuffer(GL_RENDERBUFFER, DepthRenderBuffer);
glRenderbufferStorage(GL_RENDERBUFFER, GL_DEPTH_COMPONENT, WIDTH, HEIGHT);
glBindRenderbuffer(GL_RENDERBUFFER, 0);

GLuint HDRFBO = -1;
glGenFramebuffers(1, &HDRFBO);
glBindFramebuffer(GL_FRAMEBUFFER, HDRFBO);
glFramebufferTexture2D(GL_FRAMEBUFFER, GL_COLOR_ATTACHMENT0, GL_TEXTURE_2D, ColorBufferTextures[0], 0);
glFramebufferTexture2D(GL_FRAMEBUFFER, GL_COLOR_ATTACHMENT1, GL_TEXTURE_2D, ColorBufferTextures[1], 0);
glFramebufferRenderbuffer(GL_FRAMEBUFFER, GL_DEPTH_ATTACHMENT, GL_RENDERBUFFER, DepthRenderBuffer);
GLuint Attachments[2] = { GL_COLOR_ATTACHMENT0,GL_COLOR_ATTACHMENT1 };
//显式告知OpenGL我们需要使用MRT
glDrawBuffers(2, Attachments);
if(glCheckFramebufferStatus(GL_FRAMEBUFFER) != GL_FRAMEBUFFER_COMPLETE)
    std::cout << "Framebuffer not complete!" << std::endl;
glBindFramebuffer(GL_FRAMEBUFFER, 0);

需要注意的是，其中我们需要调用glDrawBuffers函数来显式告知OpenGL我们需要使用MRT，把多个结果渲染到多个颜色缓冲里，给该函数传递的参数就是所要用到的缓冲附件的枚举数组。

那么现在我们可以去着色器里提取出明亮区域了，代码如下：

#version 430 core
layout (location = 0) out vec4 FragColor_;
layout (location = 1) out vec4 BrightColor_;
[...]
void main()
{
    [...]
    FragColor_ = vec4(Result, 1.0);
    float Brightness = dot(Result, vec3(0.2126, 0.7152, 0.0722));
    if(Brightness > 1.0)
        BrightColor_ = vec4(Result, 1.0);
    else
        BrightColor_ = vec4(0, 0, 0, 1);
}

其中Result是多个光源作用下的光照结果。我们让这个结果和一个设定好的值进行点乘，如果点乘结果大于1.0，就认为对应的颜色是比较亮的。

上面的着色器程序是适用于接受光照的场景的，我们还想要把光源也画出来，那么光源也应该有自己的一套着色器，它提取亮色的片元着色器与上面类似：

#version 430 core
layout (location = 0) out vec4 FragColor_;
layout (location = 1) out vec4 BrightColor_;

uniform vec3 u_LightColor;

void main()
{
    FragColor_ = vec4(u_LightColor, 1.0);

    float Brightness = dot(FragColor_.rgb, vec3(0.7152, 0.7152, 0.7152));
    if(Brightness > 1.0)
        BrightColor_ = vec4(FragColor_.rgb, 1.0);
    else
        BrightColor_ = vec4(0, 0, 0, 1);
}

它们都是先把每个片元的本身颜色借助于FragColor_输出到第一个颜色缓冲里，然后把比较亮的颜色借助于BrightColor_输出到第二个颜色缓冲里（其中不够亮的颜色直接重置为黑色）。

需要注意的是，如果不借助HDR的话，想要提取出明亮区域，我们就必须要把颜色值压到[0,1]之间，阈值也不能设为Brightness > 1.0了，而是要设为比1.0更小的数，但是这会很容易导致提取出来的明亮区域过多，从而导致光晕效果过重。

高斯模糊

在 OPENGL19 帧缓冲一文中，我们采用的模糊是一个图像中所有周围像素的均值，它的确为我们提供了一个简易实现的模糊，但是效果并不好。而高斯模糊是基于高斯曲线（正态分布曲线），曲线形状如下所示：

高斯曲线在它的中间处值最大，两边逐渐变小。我们借助高斯曲线的值，比如以四周像素到中间像素的距离作为x轴，取得对应的高斯曲线y值，来作为对应像素的权重（而不是像以前那样周围取均值时周围所有像素的权重都是一样的），这样可以得到更好更真实的模糊效果，即高斯模糊。

高斯模糊还有一个好处：如果我们对每一个像素用周围32x32的kernel区域来进行模糊处理，那么通常对每个片元我们都要从纹理中采样1024次！不过幸运的是，二维高斯模糊方差可以分解成两个更小的方程：现进行水平模糊，再进行垂直模糊，这样只需要采样32+32次，但是得到的结果却和采样1024次来进行全方位模糊是一样的！这叫做两步高斯模糊。如下图所示：

不过我们通常会对一个图像进行多次高斯模糊。也就是说先进行水平模糊，再进行垂直模糊，再进行水平模糊，再进行垂直模糊……，直到达到指定的高斯模糊次数为止。很容易想到我们可以借助于两个帧缓存来实现这个过程，帧缓存1进行水平模糊，把模糊结果交给帧缓存2来进行垂直模糊，然后帧缓存2把模糊结果交给帧缓存1来进行水平模糊…，如此循环反复。这两帧缓存可以被称为pingpong缓存技术。

我们先看一下高斯模糊的片元着色器怎么写：

#version 430 core

in vec2 v2f_TexCoords;
out vec4 FragColor_;

uniform sampler2D u_Image;
uniform bool u_IsHorizontal;
uniform float u_GaussianWeight[5] = float[] (0.2270270270, 0.1945945946, 0.1216216216, 0.0540540541, 0.0162162162);

void main()
{
    vec2 TexelOffset = 1.0 / textureSize(u_Image, 0); //每个纹素的大小
    vec3 Result = texture(u_Image, v2f_TexCoords).rgb * u_GaussianWeight[0];
    if(u_IsHorizontal)
    {
        for(int i = 1; i < 5; ++i)
        {
        Result += texture(u_Image, v2f_TexCoords + vec2(TexelOffset.x * i, 0.0)).rgb * u_GaussianWeight[i];
        Result += texture(u_Image, v2f_TexCoords - vec2(TexelOffset.x * i, 0.0)).rgb * u_GaussianWeight[i];
        }
    }
    else
    {
        for(int i = 1; i < 5; ++i)
        {
            Result += texture(u_Image, v2f_TexCoords + vec2(0.0, TexelOffset.y * i)).rgb * u_GaussianWeight[i];
            Result += texture(u_Image, v2f_TexCoords - vec2(0.0, TexelOffset.y * i)).rgb * u_GaussianWeight[i];
        }
    }
    FragColor_ = vec4(Result, 1.0);
}

可以看到代码很简单，根据bool值horizontal来确定当前pass是执行水平模糊还是垂直模糊，然后根据周围纹素到当前片元对应的纹素的距离，去预先指定的高斯函数y值数组weight中拿到对应的权重，将横向或者纵向上的5个纹素值进行加权求和。

创建两个pingpong帧缓存当然也很简单了，和以前一样：

GLuint PingpongColorBufferTextures[2];
glGenTextures(2, PingpongColorBufferTextures);
for (GLuint i = 0; i < 2; ++i)
{
    glBindTexture(GL_TEXTURE_2D, PingpongColorBufferTextures[i]);
    glTexImage2D(GL_TEXTURE_2D, 0, GL_RGBA16F, WIDTH, HEIGHT, 0, GL_RGBA, GL_FLOAT, nullptr);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);  // we clamp to the edge as the blur filter would otherwise sample repeated texture values!
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);
    glBindTexture(GL_TEXTURE_2D, 0);
}

GLuint PingpongFBO[2];
glGenFramebuffers(2, PingpongFBO);
for (int i = 0; i < 2; ++i)
{
    glBindFramebuffer(GL_FRAMEBUFFER, PingpongFBO[i]);
    glFramebufferTexture2D(GL_FRAMEBUFFER, GL_COLOR_ATTACHMENT0, GL_TEXTURE_2D, PingpongColorBufferTextures[i], 0);
    if (glCheckFramebufferStatus(GL_FRAMEBUFFER) != GL_FRAMEBUFFER_COMPLETE)
        std::cout << "Framebuffer not complete!" << std::endl;
    glBindFramebuffer(GL_FRAMEBUFFER, 0);
}

然后我们循环Amount次，先进行水平模糊，再进行垂直模糊，这样就能得到Amount/2次的高斯模糊结果了：

//高斯模糊
bool IsHorizontal = true, IsFirstIteration = true;
GLuint Amount = 10;
GaussianBlur_Shader.Use();
for (GLuint i = 0; i < Amount; ++i)
{
    glBindFramebuffer(GL_FRAMEBUFFER, PingpongFBO[IsHorizontal]);
    glClear(GL_COLOR_BUFFER_BIT);
    glUniform1i(glGetUniformLocation(GaussianBlur_Shader.shaderProgram, "u_IsHorizontal"), IsHorizontal);
    glActiveTexture(GL_TEXTURE0);
    glBindTexture(GL_TEXTURE_2D, IsFirstIteration ? ColorBufferTextures[1] : PingpongColorBufferTextures[!IsHorizontal]);
    RenderQuad();
    IsHorizontal = !IsHorizontal;
    if (IsFirstIteration)
        IsFirstIteration = false;
}

混合HDR纹理和模糊纹理

最后我们只需要把场景渲染得到的HDR纹理和高斯模糊之后的纹理，进行简单叠加，再做色调映射和Gamma校正就能得到最终的光晕效果了：

#version 430 core

in vec2 v2f_TexCoords;
out vec4 Color_;

uniform sampler2D u_HDRTexture;
uniform sampler2D u_BluredTexture;

void main()
{
    vec3 HDRColor = texture(u_HDRTexture, v2f_TexCoords).rgb;
    vec3 BloomColor = texture(u_BluredTexture, v2f_TexCoords).rgb;
    HDRColor += BloomColor;
    //曝光色调映射，曝光度1.0
    float Exposure = 1.0;
    HDRColor = vec3(1.0) - exp(-HDRColor * Exposure);
    //Gamma校正
    HDRColor = pow(HDRColor, vec3(1.0 / 2.2));
    Color_ = vec4(HDRColor, 1.0);
}

需要注意的是，我们要在应用色调映射之前添加泛光效果。这样添加的亮区的泛光，也会柔和转换为LDR，光照效果相对会更好。结果如下图所示：
