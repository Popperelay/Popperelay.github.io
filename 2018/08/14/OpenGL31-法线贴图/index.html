为何要使用法线贴图

我们可以使用纹理来表现物体表面的细节，比如可以给一个四边形贴上砖块纹理，它变得就很像一堵墙了：

但是仔细观察会发现，给这面墙打上光时，墙面看起来很平滑，实际上我们更想要砖块之间缝隙（以及细小的空洞、颗粒）等等看起来更明显，也就是说砖块缝隙的反光不应该像上图那么强烈。

其实上图里砖块缝隙反光之所以那么强烈，是因为整个墙面都用的是同一个法线向量（即四边形的法线向量），有没有什么办法让砖块缝隙可以使用不同的法线向量呢？一种解决方案是我们可以使用法线贴图（normal mapping）（或者叫凹凸贴图（bump mapping））。因为法线贴图里面存储了每个片元的法线，我们可以在片元着色器里从法线贴图里提取出每个片元对应的法线向量。下图是对这个墙面使用和不使用法线贴图下的光照对比：

可以看到细节得到了很明显的提升。

应用法线贴图

下面是墙面对应的法线贴图：

之所以总体上偏向蓝色，是因为上面墙面四边形的法线是(0,0,1)，那么墙面上的每个片元的法线大致都偏向于（0,0,1），所以偏蓝色。

将这个法线贴图应用于片元着色器的代码类似如下：

uniform sampler2D u_WallNormalMap;

void main()
{           
    // 从法线贴图范围[0,1]获取法线
    Normal = texture(u_WallNormalMap, v2f_TexCoord).rgb;
    // 将法线向量转换为范围[-1,1]
    Normal = normalize(Normal * 2.0 - 1.0);        //把法线纹理中[0,1]范围内的法线，变换到[-1,1]之间

    [...]
    // 像往常那样处理光照
}

因为法线贴图中存储的法线向量坐标范围是[0,1]，所以我们需要对它们先乘以2再减1.0，来变换回正常的[-1,1]的范围。

然后再使用这个法线向量，执行Blinn-Phong光照，就能得到如下图所示的结果：

上面法线贴图的限制

上面的法线贴图是怎么来的？它是对上面立起来的那面墙的每个片元，设置了一个对应的法线向量，然后存储到纹理里，从而得到对应的法线贴图。但是如果我们的墙面不是立起来的，而是躺着或是其他任意角度呢？比如下面这样：

那么我们是不是还需要为这个躺着的墙面再做一张对应的法线贴图？如果还用的是原来立起来的那张法线贴图的话，就会出现上图所示的错误光照。

仔细想想，之所以当墙面角度不同时会有不同的法线，是因为我们是在世界空间下去看待这个法线的。如果我们是以物体本身作为参照来创建法线，而不是像在世界空间下以世界坐标系作为参照来创建法线，那么就不会有上面的问题了。解决方案是我们可以在片元的切线空间里创建法线，切线空间是以片元表面的法线为z轴，以片元表面的切向量为x轴，以副切向量为y轴的坐标系空间。如下图所示：

但是片元的切向量有很多，我们应该选哪一条呢？如下图所示：

理论上来说，哪条都是可以的。但是为了避免一些麻烦，我们通常选择和纹理坐标方向一致的 那条切向量，如下图所示：

副切向量当然就可以通过法线和切向量正交（叉乘）而来了。

理论上是这样来找切向量和副切向量的，但是我们通常并不知道纹理坐标在世界空间里面对应的方向是多少，也就很难直接去确定切向量的方向。所以需要一些数学推导了。如下图所示：

图中$T$和$B$分别是对应纹理坐标方向的切向量和副切向量，三角形$\Delta P_1P_2P_3$由于处于纹理空间中（以四边形所在平面，左下角为原点），所以我们可以把边$E_1$、$E_2$用$T$和$B$表示出来（值得注意的是$T、B、E_1、E_2$都是在世界空间下的向量）：
$$E_1 = (U_1-U_2)T + (V_1-V_2)B \\
E_2 = (U_3-U_2)T + (V_3-V_2)B$$
我们也可以写成这样：
$$
(E_{1x}, E_{1y}, E_{1z}) = \Delta U_1(T_x, T_y, T_z) + \Delta V_1(B_x, B_y, B_z) \\
(E_{2x}, E_{2y}, E_{2z}) = \Delta U_2(T_x, T_y, T_z) + \Delta V_2(B_x, B_y, B_z)
$$
我们可以把上面的方程组改写成等价的矩阵形式：
$$
\begin{bmatrix} E_{1x} & E_{1y} & E_{1z} \ E_{2x} & E_{2y} & E_{2z} \end{bmatrix} = \begin{bmatrix} \Delta U_1 & \Delta V_1 \ \Delta U_2 & \Delta V_2 \end{bmatrix} \begin{bmatrix} T_x & T_y & T_z \ B_x & B_y & B_z \end{bmatrix}
$$
我们的目标是要求出$T$和$B$，那么可以等式两边乘以逆矩阵，有：
$$
\begin{bmatrix} \Delta U_1 & \Delta V_1 \ \Delta U_2 & \Delta V_2 \end{bmatrix}^{-1} \begin{bmatrix} E_{1x} & E_{1y} & E_{1z} \ E_{2x} & E_{2y} & E_{2z} \end{bmatrix} = \begin{bmatrix} T_x & T_y & T_z \ B_x & B_y & B_z \end{bmatrix}
$$
我们可以通过1除以矩阵的行列式，再乘以它的伴随矩阵，来求一个方形矩阵的逆矩阵，所以：
$$\begin{bmatrix} T_x & T_y & T_z \ B_x & B_y & B_z \end{bmatrix} = \frac{1}{\Delta U_1 \Delta V_2 - \Delta U_2 \Delta V_1} \begin{bmatrix} \Delta V_2 & -\Delta V_1 \ -\Delta U_2 & \Delta U_1 \end{bmatrix} \begin{bmatrix} E_{1x} & E_{1y} & E_{1z} \ E_{2x} & E_{2y} & E_{2z} \end{bmatrix}$$
有了这个等式，我们就可以根据三角形的两条边以及纹理坐标的差值计算出切向量$T$和副切向量$B$。

这样我们就可以把从法线贴图中获取到的法线向量，乘以矩阵$(T,B,N)$来完成从切线空间到世界空间的转换。

应用切线空间下的法线贴图

现在我们把原来的法线贴图看作是在切线空间下的（实际上确实是在切线空间下，只不过这个朝向+z的立起来的墙面，其切线空间刚好和世界空间相等而已）。

手动计算切向量和副切向量

先根据顶点坐标和纹理坐标计算出墙面的切向量和副切向量：

// positions
glm::vec3 pos1(-1.0,  1.0, 0.0);
glm::vec3 pos2(-1.0, -1.0, 0.0);
glm::vec3 pos3(1.0, -1.0, 0.0);
glm::vec3 pos4(1.0, 1.0, 0.0);
// texture coordinates
glm::vec2 uv1(0.0, 1.0);
glm::vec2 uv2(0.0, 0.0);
glm::vec2 uv3(1.0, 0.0);
glm::vec2 uv4(1.0, 1.0);
// normal vector
glm::vec3 nm(0.0, 0.0, 1.0);

glm::vec3 edge1 = pos2 - pos1;
glm::vec3 edge2 = pos3 - pos1;
glm::vec2 deltaUV1 = uv2 - uv1;
glm::vec2 deltaUV2 = uv3 - uv1;

GLfloat f = 1.0f / (deltaUV1.x * deltaUV2.y - deltaUV2.x * deltaUV1.y);

tangent1.x = f * (deltaUV2.y * edge1.x - deltaUV1.y * edge2.x);
tangent1.y = f * (deltaUV2.y * edge1.y - deltaUV1.y * edge2.y);
tangent1.z = f * (deltaUV2.y * edge1.z - deltaUV1.y * edge2.z);
tangent1 = glm::normalize(tangent1);

bitangent1.x = f * (-deltaUV2.x * edge1.x + deltaUV1.x * edge2.x);
bitangent1.y = f * (-deltaUV2.x * edge1.y + deltaUV1.x * edge2.y);
bitangent1.z = f * (-deltaUV2.x * edge1.z + deltaUV1.x * edge2.z);
bitangent1 = glm::normalize(bitangent1);  

[...] // 对平面的第二个三角形采用类似步骤计算切线和副切线

计算出来的切向量的值应该是(1,0,0)，副切向量应该是(0,1,0)，它们和法线(0,0,1)组成两两垂直的TBN矩阵。

然后我们把计算出来的切向量和副切向量传到顶点着色器里，并计算出TBN矩阵：

#version 430 core
layout (location = 0) in vec3 _Position;
layout (location = 1) in vec3 _Normal;
layout (location = 2) in vec2 _TexCoord;
layout (location = 3) in vec3 _Tangent;
layout (location = 4) in vec3 _Bitangent;

uniform mat4 u_Projection;
uniform mat4 u_View;
uniform mat4 u_Model;

out mat3 v2f_TBN;

void main()
{
    [...]
    mat3 NormalMatrix = mat3(transpose(inverse(u_Model)));
    vec3 T = normalize(NormalMatrix * _Tangent);
    vec3 B = normalize(NormalMatrix * _Bitangent);
    vec3 N = normalize(NormalMatrix * _Normal);
    v2f_TBN = mat3(T, B, N);
}

注意构造TBN矩阵的时候不要忘了乘以模型矩阵（也就是代码中的法线矩阵NormalMatrix）。

那么我们就可以用这个TBN矩阵把法线从切线空间转换到世界空间了，然后统一在世界空间里进行光照计算。不过，还有另一种选择，我们可以用TBN矩阵的逆矩阵，把其它计算光照需要用到的观察点位置、光源位置、片元世界位置等等都转换到切线空间，然后在切线空间里进行光照计算。我们分别来看一下。

在世界空间下计算光照

在世界空间下计算光照，我们需要把TBN矩阵从顶点着色器发送到片元着色器：

out vec2 v2f_TexCoord;
out vec3 v2f_FragWorldPos;
out mat3 v2f_TBN;

void main()
{
    [...]
    v2f_TBN = mat3(T, B, N);
}

在片元着色器中我们用TBN矩阵来把法线贴图中的法线从切线空间变到世界空间：

vec3 Normal;
Normal = texture(u_WallNormalMap, v2f_TexCoord).rgb;
Normal = normalize(Normal * 2.0 - 1.0);        //把法线纹理中[0,1]范围内的法线，变换到[-1,1]之间
Normal = normalize(v2f_TBN * Normal);        //将法线从切线空间转换到世界空间

然后我们就可以使用这个在世界空间里的法线Normal，来在世界空间下进行光照计算了。

在切线空间下计算光照

由于要把一些光照参数（观察向量等等）从世界空间转换到切线空间，所以我们先求出TBN矩阵的逆矩阵：

mat3 InverseTBN = transpose(mat3(T, B, N));

之所以用transpose而不是inverse函数，是因为TBN矩阵是正交矩阵，而正交矩阵的逆矩阵就是其转置矩阵，所以这里直接用转置来求逆了，这样比用inverse来求逆计算量小得多。

然后我们把光照需要用到的光源位置、视点位置、法线向量等等参数，在顶点着色器里从世界空间转换到切线空间：

out vec3 v2f_FragPosInTangentSpace;
out vec3 v2f_ViewPosInTangentSpace;
out vec3 v2f_LightPosInTangentSpace;

uniform vec3 u_ViewPos;
uniform vec3 u_LightPos = vec3(0.5, 0.5, 0.3);

[...]

void main()
{    
    [...]
    v2f_FragPosInTangentSpace = InverseTBN * FragPosInWorldSpace.xyz;
    v2f_ViewPosInTangentSpace = InverseTBN * u_ViewPos;
    v2f_LightPosInTangentSpace = InverseTBN * u_LightPos;
}

然后在片元着色器里，我们就可以使用这些已经在切线空间下的光照参数，来统一在切线空间下进行光照计算。

运行结果和上面一样。所有源码请看这里。

值得注意的是，我们并没有把TBN逆矩阵传到片元着色器，然后在片元着色器里把光照参数从世界空间转换到切线空间，而是在顶点着色器里做的这件事，效果相差很小，但是计算量会小很多，是个很好的优化。

对复杂物体应用法线贴图

我们对前面的墙面可以手动来计算切线和副切线向量，但是对于有几十万个面片的复杂模型来说，稍微有点麻烦。幸好，模型加载库Assimp已经帮我们做好了。

在我们使用Assimp从文件里面加载模型的时候，加上aiProcess_CalcTangentSpace标签，Assimp就会自动为每个加载的顶点计算出柔和的切线和副切线向量：

const aiScene* scene = importer.ReadFile(
    path, aiProcess_Triangulate | aiProcess_FlipUVs | aiProcess_CalcTangentSpace
);

然后我们可以通过下面的代码来获取Assimp计算出的切线向量（副切线向量没有必要了，因为可以在着色器里通过法线和切线向量叉乘得到）：

struct Vertex 
{
    vec3 position;
    vec3 normal;
    vec2 texCoords;
    vec3 Tangent;
};

vertex.Tangent = vec3(aimesh->mTangents[i].x, aimesh->mTangents[i].y, aimesh->mTangents[i].z);

接着需要去更新网格类Mesh，让它把切线向量传入着色器，并且正确解析它：

glVertexAttribPointer(3, 3, GL_FLOAT, GL_FALSE, sizeof(Vertex), (GLvoid*)offsetof(Vertex, Tangent));
glEnableVertexAttribArray(3);

当然，我们还必须更新我们的模型加载类，让它去把模型中的法线纹理加载进来：

//获取所有的法线纹理
for (GLuint i = 0; i < aimat->GetTextureCount(aiTextureType_HEIGHT); i++) {
    aiString str;
    //获取第i个镜面反射纹理的存储路径
    aimat->GetTexture(aiTextureType_HEIGHT, i, &str);
    string texName = str.C_Str(); //纹理名称
    string texPath = directory + '/' + texName;  //纹理路径
    bool skip = false;
    for (int j = 0; j < loadedTextures.size(); j++) {
        if (texPath == loadedTextures[j].texPath) {
            skip = true;
            textures.push_back(loadedTextures[j]);
            break;
        }
    }
    if (!skip) {
        Texture texture;
        TextureLoader textureLoader;
        texture.id = textureLoader.LoadTexture(texPath.c_str());
        texture.samplerName = "u_NormalMap" + IntToString(i);
        texture.texPath = texPath;
        textures.push_back(texture);
        loadedTextures.push_back(texture);
    }
}

其中，需要注意obj格式的模型文件，它导出的法线贴图有点不一样，Assimp用aiTextureType_NORMAL并不会去加载obj模型的法线贴图，需要用aiTextureType_HEIGHT才可以。

重正交化TBN向量

前面已经提到过，因为一个顶点可能会被多个面共享，所以Assimp通常会把算出来的多个切向量进行平均化，以获得更好更平滑的结果。但是平均化以后，TBN向量就有可能不再是相互正交的了，会导致法线贴图可能会稍稍偏移。

我们可以使用一个叫做格拉姆-施密特正交化过程（Gram-Schmidt process）的数学技巧，来对TBN向量进行重正交化，这样每个向量又会重新垂直了。我们可以在顶点着色器中这样做（数学原理其实很简单，画一下图就知道了）：

vec3 T = normalize(vec3(model * vec4(tangent, 0.0)));
vec3 N = normalize(vec3(model * vec4(normal, 0.0)));
// re-orthogonalize T with respect to N
T = normalize(T - dot(T, N) * N);
// then retrieve perpendicular vector B with the cross product of T and N
vec3 B = cross(T, N);

mat3 TBN = mat3(T, B, N)
