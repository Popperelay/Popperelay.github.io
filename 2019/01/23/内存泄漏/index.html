使用CRT库函数

我们可以在程序结束前使用CRT（C Run-Time）库函数_CrtDumpMemoryLeaks函数，来检测内存泄漏：

#define _CRTDBG_MAP_ALLOC
#include <stdlib.h>
#include <crtdbg.h>

int main()
{
    auto p = new int[3];

    _CrtDumpMemoryLeaks();  //退出时检测内存泄漏
    return 0; 
}

然后按F5（Ctrl+F5不行）运行程序在VS的output窗口会看到：

Detected memory leaks!
Dumping objects ->
{175} normal block at 0x00042630, 12 bytes long.
 Data: <            > CD CD CD CD CD CD CD CD CD CD CD CD 
Object dump complete.

其中{175}表示内存泄漏的位置，12 bytes表示12个比特位没有被释放。但是从这里无法定位到是程序哪一行导致的内存泄漏，我们可以在程序开始的时候加上库函数_CrtSetBreakAlloc(175);函数，传入的参数是output中显示的内存泄漏的位置，再次运行程序后会自动断点到auto p = new int[3];这一行，发现是它动态分配的内存没有被释放造成的内存泄漏。

如果程序有多个退出点（不止main函数里的return 0可以终止程序），一种方法是在每个退出点上都使用_CrtDumpMemoryLeaks函数，但是比较繁琐。另一种方法是在程序开始的地方加上_CrtSetDbgFlag ( _CRTDBG_ALLOC_MEM_DF | _CRTDBG_LEAK_CHECK_DF );（不用再显式调用_CrtDumpMemoryLeaks函数了），这个函数会在程序的每个退出点上都自动执行_CrtDumpMemoryLeaks函数。

使用Visual Leak Detector（VLD）插件

可以在这里下载VLD插件，双击安装即可（可以不用它自己设置的环境）。在配置好VLD的库文件以后，只需要在需要检测内存泄漏的cpp源文件包含进vld.h就可以了：

#include <vld.h>

按F5调试运行后，可以在output窗口看到类似的内存泄漏信息。

使用VS内存快照

可以使用VS诊断工具自带的快照功能，来分析两次快照之间增加的内存来自哪里。比如如下程序：

void leakTest()
{
    auto p = new int[3];
    //delete p;
}

int main()
{
    leakTest();

    return 0; 
}

如果我们想要检测leakTest函数里是否发生内存泄漏，我们可以在进入该函数之前和退出该函数之后分别进行一次快照。快照的方法是：按F5进入调试运行，在诊断工具窗口的的内存使用率里，点击摄像机图标来截取快照。然后我们可以看到如下图所示的两次快照：
