_MainTex_TexelSize的坑

不在Shader的Properties里声明_MainTex，就访问不到_MainTex_TexelSize的值，会是0：

Properties
{
    _MainTex ("Base (RGB)", 2D) = "white" {}
    ...
}

ExecuteCommandBuffer的坑

Graphics.ExecuteCommandBuffer函数必须在OnPreRender、OnPostRender、OnRenderImage等函数中运行，也就是需要和摄像机相关。在Update函数中调用很可能会出错（比如渲染不出来RT）（在OnPreRender中一样有问题，脚本挂到摄像机上了，函数运行了，但是还是渲染不出来，不知道为啥。。。）。

DepthTextureMode的坑

把m_Camera.depthTextureMode设置为DepthTextureMode.Depth有时候会得不到深度图，巨特么坑。。。需要设置为深度法线模式：

m_Camera.depthTextureMode = DepthTextureMode.DepthNormals;

RWTexture2DArray的坑1

在Unity里创建可读写的纹理数组时，需要加上寄存器以及并且在CPU端设置随机写目标。在Shader里：

RWTexture2DArray _LightmapArray : register(u1);
//...
#pragma target 5.0
_LightmapArray[uint3(ScreenPos, _LightmapIndex)] = float4(1.0, 0.0, 0.0, 1.0);

还需要注意的是，需要在Shader里把Shader Model设置成5.0，因为设置寄存器是Shader Model5.0中的特性。其中u1表示1号寄存器（这个寄存器有点像是OpenGL中的纹理绑定点，显式地表示我们要把纹理绑定到哪个寄存器上）。

然后在CPU端：

RenderTexture m_lightMapArray = new RenderTexture(1024, 1024, 24, RenderTextureFormat.ARGBFloat);
m_lightMapArray.dimension = UnityEngine.Rendering.TextureDimension.Tex2DArray;
m_lightMapArray.volumeDepth = 2;
m_lightMapArray.enableRandomWrite = true;
m_lightMapArray.filterMode = FilterMode.Bilinear;
m_lightMapArray.wrapMode = TextureWrapMode.Clamp;
m_lightMapArray.useMipMap = false;
m_lightMapArray.Create();

Graphics.SetRandomWriteTarget(1, m_lightMapArray);

值得注意的是最后用SetRandomWriteTarget函数将可读写纹理数组关联到1号寄存器上。

而普通的纹理或者纹理数组，只需要使用Shader.SetGlobalTexture或者Material.SetTexture来设置纹理就可以了，不需要显式指定寄存器。这就是Unity坑的地方，用到读写纹理数组时需要显式指定寄存器。。。（在片元着色器里使用读写纹理需要显式指定寄存器，在计算着色器里不需要）

RWTexture2DArray的坑2

Unity Shader里使用RWTexture2DArray可以写，但是不能读。也就说说下面这样是有效的：

RWTexture2DArray _Baker_LightmapArray : register(u1);
_Baker_LightmapArray[uint3(ScreenPos, _Baker_LightmapIndex)] = float4(1, 0, 0, 1)

但是下面这样是无效的，读不出来：

RWTexture2DArray _Dilate_LightmapArray : register(u2);
float4 TexelColor = _Baker_LightmapArray[uint3(ScreenPos, i)]

坑比！读需要用Texture2DArray。注：这个坑比只是在针对RWTexture2DArray<float4>才是不能读的，对于其他类型的，比如RWTexture2DArray<uint>，是可读可写的。

RWTexture2DArray的坑3

Unity在RWTextureArray上坑的一匹。。。使用Graphics.DrawMesh来跑一遍Pass时，RWTextureArray是绑不上去的。。。

Shader变量默认参数的坑

在Unity Shader的vertex函数和fragment函数之外设置变量的默认值是无效的。例如：

Pass
{
    CGPROGRAM
    float2 uvOffset = float2(100.0 / 1024, 100.0 / 1024);
    //...
    v2f vert()
    {
        //...
    }

    fixed4 frag()
    {
        //...
    }

    #pragma vertex vert
    #pragma fragment frag
    ENDCG

上面给uvOffset设置的默认值是无效的，运行起来实际上值还是0。需要到函数里去设置这个值，或者从CPU把值传过来。坑的一匹（因为内部实际上跟材质有关）~

清空数组纹理RT的坑

对一个RenderTexture设置默认颜色值，可以如下设置（类似于OpenGL中的clear）：

RenderTexture RT = UnityEngine.RenderTexture.active;
UnityEngine.RenderTexture.active = Tex;
GL.Clear(true, true, new Color(0.6f, 0.6f, 0.6f, 1));
UnityEngine.RenderTexture.active = RT;

但是如果该RenderTexture是一个纹理数组，即它的创建如下：

var Tex = new RenderTexture(width, height, 24, RenderTextureFormat.ARGBFloat);
Tex.dimension = TextureDimension.Tex2DArray;  //纹理数组类型
Tex.volumeDepth = lightMapCount;  //纹理数组长度
Tex.enableRandomWrite = true;  //可在GPU里进行读写
Tex.filterMode = FilterMode.Bilinear;
Tex.wrapMode = TextureWrapMode.Clamp;
Tex.useMipMap = true;
Tex.Create();

这时使用之前的方式，只能把RenderTexture中第一层纹理清空为默认颜色值，而无法对所有层纹理都生效。应该使用Graphics.SetRenderTarget来代替UnityEngine.RenderTexture.active：

RenderTexture RT = UnityEngine.RenderTexture.active;
for (int i = 0; i < lightMapCount; ++i)
{
    Graphics.SetRenderTarget(Tex, 0, CubemapFace.Unknown, i);
    GL.Clear(false, true, new Color(0.6f, 0.6f, 0.6f, 1.0f));
}
Graphics.SetRenderTarget(RT);

数组纹理和RT多次使用时的坑

Unity的RenderTexture（实际上是个Buffer）和读写Texture的寄存器在底层可能是有绑定关系的。如下程序，我先在第一遍pass中申请临时RT，再绑定一个渲染纹理数组（实际上是一个有深度维数的RenderTexture）到寄存器上面（这个可以参考前面的第4点）：

Graphics.SetRandomWriteTarget(2, m_dilatedLightmapArray);  //绑定渲染纹理数组到2号寄存器上

RenderTexture m_TmpTexture1 = RenderTexture.GetTemporary(1024, 1024);  //分配临时RT
RenderTexture m_TmpTexture2 = RenderTexture.GetTemporary(1024, 1024);

m_dilateLightmapMat.SetTexture("_Baker_LightmapArray", m_lightMapArray);  //绑定一个渲染纹理数组到2D纹理数组上，可以参考前面第5点
Graphics.Blit(m_TmpTexture1, m_TmpTexture2, m_Mat1);

RenderTexture.ReleaseTemporary(m_TmpTexture1);  //释放临时RT
RenderTexture.ReleaseTemporary(m_TmpTexture2);

Graphics.ClearRandomWriteTargets();  //清理寄存器

然后在第二个Pass里，我又申请了临时RT，再绑定渲染纹理数组到寄存器上面：

RenderTexture m_TmpTexture1 = RenderTexture.GetTemporary(1024, 1024);
RenderTexture m_TmpTexture2 = RenderTexture.GetTemporary(1024, 1024);

Graphics.SetRandomWriteTarget(3, m_testRenderTextureArray);  //这是一个新的渲染纹理数组

Graphics.Blit(m_TmpTexture1, m_TmpTexture2, m_Mat2, 0);
Graphics.ClearRandomWriteTargets();

然后发现渲染纹理数组m_testRenderTextureArray死活绑定不到GPU上。如下图所示：

而第一个Pass里，渲染纹理数组是绑定上去了的，如下图所示：

但是看两个图里面的第一行数据都是Render Target View 196573，说明这两个Pass分配的临时渲染纹理实际上是同一个，然后查了下官方文档，确实是这样，申请的临时内存连续几帧不用才会被释放，而我这两个Pass是在一帧内运行的，导致第二个Pass实际上复用了第一个Pass申请的临时渲染纹理内存，也就是说它俩实际上是同一个内存对象。所以猜测导致第二个Pass绑定不上渲染纹理数组是因为，寄存器和渲染纹理（渲染纹理实际上应该是个帧缓存对象）是有绑定关系的，第一个临时渲染纹理在释放以及调用ClearRandomWriteTargets清理寄存器时，并没有正确释放寄存器和渲染纹理之间的绑定关系，导致第二次再用到相同的渲染纹理时，无论如何都绑定不上任何寄存器了。

然后改了下代码，第二个Pass不使用临时渲染纹理，而是正常分配的渲染纹理，就成功地把渲染纹理数组绑定上去了：

RenderTexture m_TmpTexture1 = new RenderTexture(1024, 1024, 0, RenderTextureFormat.ARGB32);  //正常分配渲染纹理数组，非临时内存
m_TmpTexture1.Create();
m_TmpTexture1.hideFlags = HideFlags.HideAndDontSave;

RenderTexture m_TmpTexture2 = new RenderTexture(1024, 1024, 0, RenderTextureFormat.ARGB32);
m_TmpTexture2.Create();
m_TmpTexture2.hideFlags = HideFlags.HideAndDontSave;

Graphics.SetRandomWriteTarget(3, m_lightMapArray);
m_blurLightmapMat.SetTexture("_Baker_LightmapArray", m_dilatedLightmapArray);
Graphics.Blit(m_TmpTexture1, m_TmpTexture2, m_blurLightmapMat, 0);
Graphics.ClearRandomWriteTargets();

ReleaseRenderTexture(ref m_TmpTexture1);
ReleaseRenderTexture(ref m_TmpTexture2);

//....
private void ReleaseRenderTexture(ref RenderTexture renderTexture)
{        
    renderTexture.DiscardContents();
    renderTexture.Release();
    Object.DestroyImmediate(renderTexture);
    renderTexture = null;
}

同理，如果想要在GPU上对渲染纹理数组作高斯模糊，需要在循环里不断创建新的RT，而不能复用，否则绑不上：

for (int i = 0; i < Iterations; ++i)
{
    RenderTexture m_TmpTexture1 = new RenderTexture(1024, 1024, 0, RenderTextureFormat.ARGB32);
    m_TmpTexture1.Create();
    m_TmpTexture1.hideFlags = HideFlags.HideAndDontSave;

    RenderTexture m_TmpTexture2 = new RenderTexture(1024, 1024, 0, RenderTextureFormat.ARGB32);
    m_TmpTexture2.Create();
    m_TmpTexture2.hideFlags = HideFlags.HideAndDontSave;

    Graphics.SetRandomWriteTarget(3, m_lightMapArray);
    m_blurLightmapMat.SetTexture("_Baker_LightmapArray", m_dilatedLightmapArray);
    Graphics.Blit(m_TmpTexture1, m_TmpTexture2, m_blurLightmapMat, 0);
    m_blurLightmapMat.SetTexture("_Baker_LightmapArray", null);
    Graphics.ClearRandomWriteTargets();

    ReleaseRenderTexture(ref m_TmpTexture1);
    ReleaseRenderTexture(ref m_TmpTexture2);

    m_TmpTexture1 = new RenderTexture(1024, 1024, 0, RenderTextureFormat.ARGB32);
    m_TmpTexture1.Create();
    m_TmpTexture1.hideFlags = HideFlags.HideAndDontSave;

    m_TmpTexture2 = new RenderTexture(1024, 1024, 0, RenderTextureFormat.ARGB32);
    m_TmpTexture2.Create();
    m_TmpTexture2.hideFlags = HideFlags.HideAndDontSave;

    Graphics.SetRandomWriteTarget(3, m_dilatedLightmapArray);
    m_blurLightmapMat.SetTexture("_Baker_LightmapArray", m_lightMapArray);
    Graphics.Blit(m_TmpTexture1, m_TmpTexture2, m_blurLightmapMat, 1);
    m_blurLightmapMat.SetTexture("_Baker_LightmapArray", null);
    Graphics.ClearRandomWriteTargets();

    ReleaseRenderTexture(ref m_TmpTexture1);
    ReleaseRenderTexture(ref m_TmpTexture2);
}

UnityEditor和MonoBehaviour的坑
